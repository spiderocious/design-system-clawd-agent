# Solon — end-of-project notes (2026-05-11)

## What worked

- **Stance picked itself.** When the user accepted all four reference picks (Economist + Bloomberg + INEC + Atlas Obscura), the answer was Editorial broadsheet (#2) with the parliamentary/atlas ornament dialled up on exports and the Bloomberg-instrument idiom dialled up on the War Room. The name *Solon* gave the stance its moral centre — wisdom + neutrality + parliamentary deliberation.

- **The mono-fact / serif-projection split.** The user's instinct was the key. Splitting numeric type by semantic role (counted fact vs modelled estimate) is the single most defensible decision in the system — it means a reader can tell at a glance whether something is signed and counted, or projected. This wasn't a "type choice"; it was an *information-architecture* choice expressed through type. Worth keeping as a Solon-specific principle.

- **Four hero scenes worked because they share a chrome.** Simulator builder, constituency map, War Room, finance dashboard — each is a different operational register, but they share masthead, sidebar idioms, the AI copilot card, the chip vocabulary, and the disclosure footnoting. The system held without forcing.

- **Reserving oxblood for irreversible.** Three tiers — forest for action, orange for warn, oxblood for irreversible — gives each colour real semantic load. The CRITICAL modal (Void result · type VOID) is the only place oxblood appears at scale, which is exactly how it should be.

## What pushed back

- **Light-only War Room.** The user picked light-only over a War Room dark variant. Mid-build I felt the tension — an election-night room in a projector hall would benefit from dark — but the user's call was probably right for v1. The ink-on-cream War Room with the inverted counts strip carries enough operational gravity that dark mode feels like an enhancement, not a requirement. **Revisit if the user reverses after seeing the WR in practice.**

- **The five-tier chip system on the polling-unit list.** Tutorly's chip system was 3 tiers; Solon needs 5 (Strong hold / Lean hold / Toss-up / Lean opp. / Strong opp.). The filled-chip approach (B's variation) carries that count better than A's outlined chips would have — five outlines start to feel like a colour wheel. The user's mixed pick (A base + B chips) was the right call.

## Decisions to remember next session

- **The S. drop-cap with an orange period** is the Solon brand mark. It's the only decorative use of orange in the system. Keep it sacred — don't sprinkle orange dots elsewhere.
- **Lever scrubbers are bidirectional from a centre tick.** This wasn't obvious before building; it became obvious when the levers needed to express both "+ 8pt youth turnout" and "- 8pt youth turnout" in the same control. The centre tick is the baseline, the fill grows in either direction. Keep this for any future "scenario lever" idiom.
- **Confidence is always a 5-segment bar plus a written hedge** (`Medium-high · n=412 PU`). Never just a stoplight. The bar gives the magnitude, the n-count gives the basis.
- **The INEC export sheet exists as a real screen, not just a print stylesheet.** It's a destination, a viewable artefact, with a drop-cap intro and a signed signature block. Treat statutory exports as designed surfaces in compliance-heavy domains — they're not afterthoughts.

## What I'd warn the implementer (`/ship-design-system`)

- The mono/serif numeric split must survive into the React library. If a developer adds a generic `<Number>` component that picks one font, the semantic rule is dead. Encode it as **two** components: `<RecordNumber>` (mono, tabular, takes a count) and `<ProjectionNumber>` (serif, takes a value + confidence band). Don't let them collapse.
- The lever scrubber is a *bidirectional centred range slider*. Standard HTML range inputs are unidirectional. Likely needs a custom React component.
- The drop-cap on the AI copilot body is `::first-letter` — works in CSS, but if the body is rendered as multiple React text nodes, the first-letter pseudo will apply only to the first node. Implementer should render the body as a single string with citations as `<sup>` tags within it.
- Win-probability chips should be a typed prop set, not a freeform label: `tier: 'strong-hold' | 'lean-hold' | 'toss-up' | 'lean-opp' | 'strong-opp'`. The colour and label both derive from the tier.
